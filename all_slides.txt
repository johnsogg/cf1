## Slide 6

Variables
Give names to your favorite things
A variable is a symbol that has a value, like:
mouseX: the horizontal coordinate of the mouse
Functions are also symbols, but generally we use them by invoking them:
const x = someFunction(someArgument);
Here we invoke someFunction using someArgument as input, and capture the result into the variable that we call x
‘const’ means “it should never change”. If you want to put different values into the variable, declare it with ‘let’

## Slide 7

Variables
Give names to your favorite things
‘const’ means “it should never change”. 
If you want to put different values into the variable, declare it with ‘let’:
let x = someFunction(someArgument)
In some older sources on the web might use ‘var’. Don’t use var. It is a holdover from 1995, and we don’t want any further reminders of the 90s.

## Slide 8

Declare -> Initialize -> Use
And when do we do these things?
Declare a variable with let (or const), followed by the name:

## Slide 9

Declare -> Initialize -> Use
And when do we do these things?
Initialize a variable by putting a value into it for the first time.
Typically this is done on the same line that you declare it.
let x = 2 * 4; // There is a variable named x and its value is 8

## Slide 10

Declare -> Initialize -> Use
And when do we do these things?
Use the variable  after  you’ve declared and initialized it
If you use it before it is declared, you’ll get errors
If you use it before it is initialized, it will have a default value that is probably not what you want.
In this case, x is 8, so we put the circle at (8, 10) with a radius of 20.
circle(x, 10, 20); // draws a circle at coordinate (x, 10)

## Slide 21

Modulo what?
Remember long division?
What’s 18 divided by 4? A fifth grader would probably tell you: 4, remainder 2.
In code,  integer division like 18 / 4 actually equals 4
We can use the modulo operator to get at the remainder.
It is the percent sign, and looks like this:
18 % 4
So if x = 18 % 4, then x will then have the value 2, because the remainder is 2.

The stuff inside the curly braces is the code that will run  only if the condition is met .  It can be a single statement, or it could be something complicated. Could even be empty!
In our example here, we just set the fill color.

## Slide 11

if (mouseX > windowWidth / 2) {
    fill("red");
} else {
    fill("blue");
}
This is an augmented form of an if-statement that has an  else  clause. If the boolean expression is not met, we run the curly braced code following the  else  keyword.
In this case, we set the fill color to blue.

## Slide 12

if (mouseX > windowWidth / 2) {
    fill("red");
} else {
    fill("blue");
}
One nice thing about structuring code like this is that you can be sure that  exactly one  of the clauses will run. So if you know you must set the fill color to either red or blue, this is a good way to do it.

## Slide 13

if (mouseX > windowWidth / 2) {
    fill("red");
}  else if (mouseY > windowHeight / 2) {
    fill("red");
} else {
    fill("blue");
}
An if-statement can also have “else if” clauses, like the one bolded here. This will run only if a previous conditional clause didn’t match.
The idea in this code is to set the fill color to red if the mouse is either in the right half, or the bottom half of the canvas. Otherwise, it sets the fill color to blue.

## Slide 14

    fill("red");
} else if (mouseY > windowHeight / 2) {
    fill(“red");
} else if (somethingElse) {
    fill(“green”);
} else {
    fill("blue");
}
You can have as many  else-if  clauses as you want, but they have to appear after the initial  if  clause, and before the  else  clause, if there is one.

It is also possible to have an if-statement that contains an “else if”, but doesn’t have a final “else” statement. Use this if you don’t want to have default behavior that is run if none of the given conditions pass.
E.g. in this code, we might not even change the fill color.

## Slide 18

Boolean Expressions
True and False, and Truthy and Falsy
A boolean expression is code that evaluates to true or false: “is it raining?” Or “is the mouse X value greater than 50?” These are yes/no questions. We’ve been looking at one that looks like this:
if ( mouseX > windowWidth / 2 ) { … }

## Slide 19

Boolean Expressions
True and False, and Truthy and Falsy
There are many other forms a boolean expression can take.
if ( isMousePressed ) { … }
There’s no equality operator, no greater-than-or-equal, etc. Variables can be interpreted as true or false on their own, if it makes sense.

## Slide 20

Boolean Expressions
True and False, and Truthy and Falsy
Since p5.js is just Javascript, we inherit all the weirdnesses of that language.
One major weirdness (that is also extremely useful) is how it can interpret different values as being true or false, even if the data it is examining isn’t even boolean. When we do this, we say that the expression is  truthy  or  falsy  - these are actual words and I am not making them up!

## Slide 23

Boolean Expressions
Combining with And and Or
Use these two operators to combine individual boolean expressions to make more sophisticated expressions. Be careful to not be too clever, because it can be hard to read.
if (mouseX > windowWidth / 2  ||  mouseY > windowHeight / 2) {
  // this code runs if the mouse is in the right half,
  // or if it is on the bottom half.
}
if (mouseX > windowWidth / 2  &&  mouseY > windowHeight / 2) {
  // this code runs if the mouse is in the lower right corner
}

Boolean Expressions
Combining with And and Or
Beware of this land mine! When you have a boolean expression that involves the  or  operator, the language will be “smart” and stop evaluating the expression when it knows the answer.
In this example, say that thingA() returns false. It will then run thingB() - which in this story, returns true. Javascript now knows that this whole expression is true, so it won’t even invoke thingC().
if (thingA() || thingB() || thingC()) { ... }

## Slide 26

Boolean Expressions
Combining with And and Or
There is a similar land mine for  and  expressions: it has to evaluate everything to know if the expression is true, but it only has to find a single false before it knows that the expression is false.
if (thingA() && thingB() && thingC()) { ... }

node myAwesomeScript.js
If your program is a non-trivial application, you’ll probably run via a tool like npm. For example, to run a web server that hosts a React app you could run:
npm run dev

## Slide 7

The runtime environment is Python:  https://www.python.org/
It is used to read Python code and interpret it
Python lacks a built-in package manager. You can install one such as pip, conda, uv, and (the one I like) poetry.
Use poetry (or others) to initialize a project, give it structure, install and manage 3rd party dependencies.
E.g. pytorch is a popular AI toolkit and can be installed on CLI with:
poetry add torch

python my_awesome_ml_thing.py
If your program is a non-trivial application, you’ll possibly run via a tool like poetry. For example, to run a machine learning app:
poetry run app

## Slide 9

Python, Javascript, Lua, GDScript, Ruby… these are all  interpreted
This means the code does not run directly on your host computer, but requires an interpreter to translate the high level language into a format that can actually execute on your machine
This is very flexible, and can be made to be very fast (or, fast enough)
The interpreter translates high level code to machine code for you.

## Slide 10

Compiled Code
C, C++, Swift, Rust, Golang: these are all  compiled  languages
This means the code requires a compiler to translate the high level language into a binary format.
The intermediate binary format can be combined to form an executable, which is completely independent of the original high level code
You can then share the executable without sharing the code, and it will work on whatever platforms it was compiled for. Business loved this for a long time.

## Slide 9

Subclassing
The “is a” relationship is extremely common:
Student  is a  Person
CTD Major  is a  Student
Business Major  is a  Student
House Painter  is a  Person
BUT!  This “is a” structure can break down very quickly. Students also have jobs, like painting houses. So modeling it as above (even though the  is a  relationships are ‘correct’) can cause you problems.

## Slide 10

Subclassing & Composition
The “is a” approach is ubiquitous. Subclassing isn’t the only way to code it.
You can ‘compose’ a class’s abilities by simply establishing members that provide a given behavior. 
Example: somebody is a CTD major, a house painter, and a DJ. With composition, you only need a Person object composed of properties like:
‘academic_major’ (which can be None in case they’re not a student)
‘profession’ (also can be None)
‘avocation’ (same)

## Slide 11

Why I’m saying this
Object Oriented Programming has been  so popular  at points in the past that many think that OOP is the only way to model things
Languages like Java are 100% based on objects for (nearly) everything
This can blind you to other approaches
Speaking of other approaches:
Functional programming: data & behavior separated. Compute by transforming data with functions. Great for algorithms.
Declarative programming: state requirements via configuration. E.g. HTML.
Logic/constraint programming: state the rules, let result(s) fall out of that. 

## Slide 12

Why I’m saying this
The approach to writing code can be heavily influenced by your choice of language, framework, libraries.
But even within a language you likely have control over which programming paradigm you use.
Example: I write a lot of functional code in context of declarative syntax using Javascript, a language with high level support for OOP
So, be open to different paradigms. There is no one true way! Often one paradigm is great for a given problem but terrible at another.
That said: OOP is usually pretty good for modeling interactive apps & games.
